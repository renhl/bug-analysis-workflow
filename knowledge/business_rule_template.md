## 业务规则名称

<!-- 规则名称 -->

订单状态流转规则

## 规则描述

<!-- 规则的详细描述 -->

订单状态必须按照预定义的状态机流转，不允许跳跃或逆向流转。

## 状态定义

| 状态值 | 状态名 | 含义 |
|--------|--------|------|
| 1 | 待支付 | 订单已创建，等待用户支付 |
| 2 | 已支付 | 支付成功，等待发货 |
| 3 | 已发货 | 商品已发出 |
| 4 | 已完成 | 用户已确认收货 |
| 5 | 已取消 | 订单取消 |

## 状态流转规则

```
创建订单 → 待支付(1)
         ↓
      支付成功 → 已支付(2)
         ↓          ↘
      发货成功 → 已发货(3)   支付失败/超时 → 已取消(5)
         ↓
      确认收货 → 已完成(4)
```

## 状态更新条件

### 从待支付 → 已支付

**触发条件**: 支付成功回调

**前置检查**:
- payment.status = success
- payment.amount = order.amount

**更新规则**:
- order.status = 2
- 必须在事务中完成

**失败处理**:
- 如果更新失败，触发重试（最多3次）
- 如果重试失败，发送告警

### 从已支付 → 已发货

**触发条件**: 发货操作

**前置检查**:
- order.status = 2
- inventory.count >= order.quantity

**更新规则**:
- order.status = 3
- 同时扣减库存

## 异常场景处理

| 场景 | 预期处理 |
|------|---------|
| 支付成功但状态未更新 | 自动补偿更新 |
| 重复支付回调 | 忽略，保持原状态 |
| 库存不足发货 | 拒绝发货，状态不变 |

| 服务 | 职责 |
|------|------|

| payment-service | 支付处理 |
| payment-callback | 支付回调处理 |
| inventory-service | 库存扣减 |
